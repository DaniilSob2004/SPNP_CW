Task chaining. Continuations.
Нитки кода. Продолжение.
  
Задачи, как объекты, позволяют указать другие задачи, которые будут выполняться после того, как будет завершена данная.
Другая задача наз. - продолжением первой задачи.
Создание цепей из нескольких продолжений составляют "нитку"(chain) -
это есть отдельный стиль в ассинхронном программировании.

!!! Внутрение задачи не видят диспетчер, поэтому нужно указывать Dispatcher.Invoke()

// Если использовать await
----1-----====2===
%%%3%%;;;;4;;;;;

start 1              ----1-----====2===
start 3              %%%3%%    ;;;;4;;;;;
await 1, start 2              |
await 3, start 4     await 1 завершиться тут -- тогда как (3) закончился раньше (будет ожидание)

// Если поменять местами выполнение, то ожидания не будет
await 3, start 4    %%%3%%;;;;4;;;;;
await 1, start 2    ----1-----====2===
                          |   | 
                    await 3   await 1
